# Sales Intelligence Chatbot — Design Document

## Understanding Summary

- **What:** Persona-aware sales intelligence chatbot — Python + Chainlit UI, Claude API (haiku/sonnet adaptive routing), SQLite via `mcp-server-sqlite`, two materially different personas enforced in system prompts, formatted answers, multi-turn context
- **Who (Sales Rep):** Scoped to assigned region + brands, last 3 months, direct/actionable answers, tables/bullets, "I don't have that data" for out-of-scope
- **Who (Country Head):** All regions/brands, 1–3 year trends, strategic/comparative narrative, summaries + comparisons
- **Non-goals:** No real authentication, no cloud deployment, no real-time data updates

## Assumptions

1. Python 3.10+ available locally
2. Anthropic API key available
3. `uvx` available to run MCP server
4. Single local session, no multi-user concurrency
5. "Last 30–90 days" for Sales Rep maps to most recent 3 months in dataset (data ends 2024)

## Persona System

### Sales Rep System Prompt (parameterized)
```
You are a sales assistant for {rep_name}, a Sales Rep covering the {region} region.

SCOPE RULES (enforce strictly):
- Only answer questions about {rep_name}'s data in the {region} region
- Only look at the last 3 months of available data (most recent year/month combos)
- If asked about other regions or other reps, respond: "I don't have that data."

ANSWER STYLE:
- Be direct, specific, and actionable
- Lead with the number, then context
- Use tables for multi-row comparisons, bullets for lists
- Always show target vs actual and achieved_pct when discussing performance

You have access to a sales database. Use the query tool to answer questions.
Never invent numbers — only report what the database returns.
```

### Country Head System Prompt
```
You are a strategic sales analyst with full visibility across all regions and brands.

SCOPE RULES:
- Access all regions: North, South, East, West
- Access all brands: NovaBev, PureLeaf, FrostDrink, ZenWater, BoostFuel
- Default time horizon: last 3 years (2022–2024), show trends
- For trend questions, compare year-over-year

ANSWER STYLE:
- Strategic, comparative, and narrative
- Identify patterns, risks, and growth opportunities
- Use tables for regional/brand comparisons, summaries for trends
- Always contextualise a number against the broader picture

You have access to a sales database. Use the query tool to answer questions.
Never invent numbers — only report what the database returns.
```

### Session State (cl.user_session)
```python
{
  "persona": "sales_rep" | "country_head",
  "rep_name": "Alice Sharma",   # sales_rep only
  "region": "North",            # sales_rep only
  "system_prompt": "...",       # built at login
  "history": [],                # sliding window, last 10 turns
  "mcp_client": <MCPClient>     # persistent MCP connection
}
```

## MCP Client (`mcp_client.py`)

- Uses `mcp` Python package (`StdioServerParameters` + `ClientSession`)
- Spawns `mcp-server-sqlite` subprocess once per chat session
- Exposes:
  - `get_tools()` → list of tool defs to pass to Claude
  - `call_tool(name, args)` → executes tool, returns result string
- Only tool used: `query` (read-only SELECT)

---

## Agent Loop (`agent.py`)

```
1. Build messages:
   [system_prompt] + [last 10 turns] + [new user message]

2. Select model:
   - Complexity signals: trend/compare/growth/risk keywords + message length
   - ≥ 2 signals → claude-sonnet-4-5
   - Otherwise → claude-haiku-3-5

3. Call Claude API with MCP tools

4. If tool_use in response:
   a. Execute SQL via mcp_client.call_tool()
   b. Append tool_result to messages
   c. Call Claude again (max 3 iterations)

5. Return final text to Chainlit

6. Append (user + assistant) to history, cap at 10 turns
```

## Chainlit UI (`app.py`)

### Flow
```
@cl.on_chat_start
  → AskActionMessage: "Sales Rep" | "Country Head"
  → if Sales Rep: AskActionMessage with rep names from DB
  → auto-resolve region: SELECT DISTINCT region WHERE sales_rep = ?
  → build system prompt
  → store in cl.user_session
  → send welcome message with persona badge

@cl.on_message
  → retrieve session state
  → agent.run(message, session) with streaming
  → update history
```

### Persona Badge
```
🟢 Country Head                    ← all regions, strategic scope
👤 Alice Sharma  |  North Region   ← scoped to rep + region
```

### Answer Format Guide (in system prompt)
| Query type | Format |
|---|---|
| Single metric | Bold number + 1-line context |
| Multi-channel/brand breakdown | Markdown table |
| Trend over time | Bullets with YoY delta |
| Risk/strategic | Narrative + supporting table |
| Out-of-scope | "I don't have that data." |

## Token Cost Estimate

**Per conversation (10 turns, 80% haiku / 20% sonnet):**

| Model | Turns | Input | Output | Cost |
|---|---|---|---|---|
| claude-haiku-3-5 (80%) | 8 | 16,000 tok | 2,400 tok | $0.023 |
| claude-sonnet-4-5 (20%) | 2 | 5,000 tok | 1,000 tok | $0.030 |
| **Total per conversation** | | | | **~$0.053** |
| **50 DAU/day** | | | | **~$2.65/day** |
| **Monthly** | | | | **~$79/month** |

**Context window per query:** ~2,000 tokens (system prompt ~200 + history ~1,500 + tool result ~300 + user message ~30)

## Security Decisions

| Decision | Implementation |
|---|---|
| API key | `.env` + `python-dotenv`, excluded from git |
| Data sent to Claude | SQL results only (aggregates/numbers) |
| PII | Rep names are fictional — no real PII |
| DB access | Read-only via MCP `query` tool |
| Deployment | Local only, localhost, no network exposure |

## Decision Log

| Decision | Alternatives | Reason |
|---|---|---|
| Chainlit | Streamlit | Purpose-built chat UI, better product feel, built-in session state |
| Manual MCP tool loop | Anthropic beta MCP SDK, LangChain | Most transparent, best build log story, full control over context |
| Local SQLite MCP | Supabase | Matches spec exactly, simpler setup, cleaner Loom demo |
| Claude API | OpenAI | Assignment context + tool use for structured data Q&A |
| Haiku/Sonnet adaptive routing | Single model | Real cost estimate with two tiers, deliberate AI design signal |
| Sliding window (last 10 turns) | Full history | Prevents token cost growth, addressable in build log |
| Persona enforced in system prompt | UI label only | Rubric explicitly grades this |
| `.env` + `.gitignore` | Hardcoded keys | Security baseline, callout in build log |
